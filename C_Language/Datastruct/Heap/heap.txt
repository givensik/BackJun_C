힙
이진트리이되 완전 이진 트리, 모든 노드에 저장된 값(우선순위)을 자식 노드에 저장된 값보다 크거나 같아야한다. => 루트 노드에 저장된 값이 가장 커야한다.

힙의 데이터 저장 과정(최소 힙 기준)
새로운 데이터 -> 우선순위가 제일 낮다는 가정으로 데이터를 추가 
-> 부모 노드와 우선순위를 비교해서 위치를 바꾸기(마지막 위치는 노드를 추가한 이후에도 완전 이진 트리가 유지되는 마지막 레벨의 가장 오른쪽 위치)

배열로 힙을 구현 
	왼쪽 자식 노드의 인덱스 값 = 부모 노드의 인덱스 값 * 2
	오른쪽 자식 노드의 인덱스 값 = 부모 노드의 인덱스 값 * 2 + 1
	부모 노드의 인덱스 갑수 = 자식 노드의 인덱스 값 / 2
